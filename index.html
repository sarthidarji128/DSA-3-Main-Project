Key insight

Dijkstra minimises the distance from a single source to every node independently. When you union all those shortest-paths you inevitably create cycles — meaning duplicate cable runs and unnecessary infrastructure cost.

Prim's MST directly minimises total cable laid across the campus with exactly n − 1 edges for n nodes — no cycles, no redundancy, guaranteed global optimum.

Decision rule
How to choose the right approach
✅ Use incremental when:
New building connects to only one existing cluster — acts as a leaf node at the campus edge. Safe, fast, O(k log n).
⚠ Use full recompute when:
New building bridges two or more blocks — could replace a heavier existing MST edge. Full correctness required.
